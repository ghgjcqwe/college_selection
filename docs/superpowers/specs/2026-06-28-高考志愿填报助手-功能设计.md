## 一、项目概述

### 1.1 项目目标
为高考考生和家长提供一个简单易用的网页工具，帮助用户：
- 根据高考分数快速匹配适合的大学
- 通过分类浏览和兴趣测试发现适合的专业

### 1.2 目标用户
高考考生、考生家长

### 1.3 技术方案
- **前端框架**：Vue 3 + Vite
- **数据方式**：内置模拟数据（JS 文件）
- **运行方式**：纯前端，浏览器打开即可使用
- **高考模式**：通用版（按分数+位次匹配，不区分省份）

## 三、模块一：分数找学校

### 3.1 功能清单

| 序号 | 功能点 | 详细说明 | 优先级 |
|------|--------|----------|--------|
| 1.1 | 分数输入 | 用户输入高考总分（必填） | P0 |
| 1.2 | 位次输入 | 用户输入全省位次（选填） | P1 |
| 1.3 | 一键匹配 | 点击按钮，根据分数推荐学校 | P0 |
| 1.4 | 分档展示 | 结果分三档：冲刺、稳妥、保底 | P0 |
| 1.5 | 学校卡片 | 每张卡片显示：校名、城市、类型、最低分 | P0 |
| 1.6 | 学校详情 | 点击卡片进入详情页，查看完整介绍 | P0 |

### 3.2 匹配算法（简化版）

根据用户输入的分数，与学校最低录取分对比：

| 档次 | 条件 | 说明 |
|------|------|------|
| 冲刺 | 学校最低分 ≥ 用户分数 且 学校最低分 - 用户分数 ≤ 20 | 分数接近，有希望冲上 |
| 稳妥 | 用户分数 > 学校最低分 且 用户分数 - 学校最低分 ≤ 30 | 大概率能上 |
| 保底 | 用户分数 - 学校最低分 > 30 | 肯定能上，用来兜底 |

### 3.3 学校数据结构

```javascript
{
  id: 1,
  name: '清华大学',
  type: '综合类',     // 综合类/理工类/师范类/医药类/财经类 等
  city: '北京',
  tags: ['985', '211', '双一流'],
  minScore: 680,      // 最低录取分数
  description: '学校简介...',
  majors: ['计算机科学与技术', '软件工程', '电子信息工程'] // 特色专业
}
```

### 3.4 数据规模
- 预置 20-30 所大学
- 覆盖不同分数段（500分 ~ 680分）
- 覆盖不同类型（综合、理工、师范、医药、财经、政法等）

## 四、模块二：专业选择指南

### 4.1 功能清单

| 序号 | 功能点 | 详细说明 | 优先级 |
|------|--------|----------|--------|
| 2.1 | 学科分类浏览 | 按工学、医学、文学等大类分类展示 | P0 |
| 2.2 | 专业列表 | 点击分类，展开该分类下的专业卡片 | P0 |
| 2.3 | 专业详情 | 点击卡片，查看专业详细介绍 | P0 |
| 2.4 | 兴趣测试入口 | 选专业页顶部大卡片，点击进入测试 | P0 |
| 2.5 | 答题流程 | 8道选择题，逐题作答，显示进度 | P0 |
| 2.6 | 测试结果 | 根据答案推荐 3 个匹配度最高的专业 | P0 |
| 2.7 | 结果跳转 | 推荐专业可直接跳转详情页 | P0 |
| 2.8 | 再测一次 | 结果页提供重新测试按钮 | P1 |

### 4.2 专业数据结构

```javascript
{
  id: 1,
  name: '计算机科学与技术',
  category: '工学',      // 所属学科大类
  courses: ['高等数学', 'C语言', '数据结构', '算法', '操作系统', '计算机网络'],
  careers: ['软件开发工程师', '算法工程师', '产品经理', '考研深造'],
  fitPerson: ['逻辑思维强', '喜欢摆弄电脑', '数学好', '有耐心'],
  tags: ['热门', '高薪', '工科', '逻辑型', '实践型'] // 兴趣标签，用于匹配
}
```

### 4.3 学科大类（共 8 个）
1. 工学类
2. 医学类
3. 文学类
4. 经济学类
5. 管理学类
6. 教育学类
7. 理学类
8. 法学类

### 4.4 数据规模
- 预置 20-30 个专业
- 每个大类 2-5 个专业
- 覆盖热门专业和代表性专业

### 4.5 兴趣测试算法（简化版）

**测试题设计：** 共 8 道题，每题 4 个选项，每个选项对应 1-2 个兴趣标签。

**兴趣标签列表：**
- 实践型（动手能力强）
- 逻辑型（善于思考推理）
- 社交型（喜欢与人打交道）
- 艺术型（有创造力、审美）
- 研究型（喜欢钻研探索）
- 管理型（有组织领导能力）

**算法流程：**
1. 每道题选一个选项，对应标签 +1 分
2. 答完 8 题后，统计每个标签的总分
3. 取得分最高的 3 个标签
4. 去专业库中找匹配这些标签的专业，按匹配度排序
5. 展示匹配度最高的 3 个专业

## 七、开发优先级

### 第一阶段（MVP，最小可用版本）
1. 搭建项目框架（Vue + Vite + 路由）
2. 准备学校和专业的模拟数据
3. 实现首页
4. 实现「找学校」完整流程（输入 → 推荐 → 详情）
5. 实现「专业分类浏览」完整流程（分类 → 列表 → 详情）

### 第二阶段（完善功能）
6. 实现兴趣测试（答题 + 结果推荐）
7. 优化界面样式和交互体验
8. 添加页面过渡动画

## 八、设计原则

1. **新手友好**：代码结构清晰，注释详尽，方便理解和修改
2. **数据内置**：所有数据直接写在 JS 文件中，不需要数据库
3. **纯前端**：不需要后端服务器，浏览器打开即用
4. **渐进增强**：先做核心功能，再逐步扩展
5. **界面简洁**：减少干扰元素，突出核心操作
